## ‚öôÔ∏è Command-Line Interface (CLI)

In addition to the graphical interface, MazeSolver includes a fully functional CLI utility: `mazegen`, designed for scriptable generation and export of mazes.

### üîß Features

* Generate mazes using any of 6 algorithms
* Output formats: `ascii`, `json`, or `npy`
* Optional metadata output
* Customizable ASCII styles

### ‚úÖ Usage

```bash
# Install locally in development mode
pip install -e .

# Run CLI from source (safe and portable)
python -m mazegen.cli --algo recdiv --size 31 --format ascii

# Optional alias for convenience
alias mazegen="python -m mazegen.cli"
```

### üìú Arguments

* `--algo`: Algorithm name (`dfs`, `prims`, `wilsons`, `recdiv`, `handk`, `kruskals`)
* `--size`: Maze width/height (must be odd). Logical size of the maze (NxN); actual output will be (2N+1)√ó(2N+1) to include walls
* `--format`: Output format: `ascii`, `json`, or `npy`
* `--output`: Save output to file (optional)
* `--style`: For ASCII format ‚Äî `blocks`, `dots`, `unicode`, `roguelike`
* `--save-meta`: Store generation metadata as `.meta.json`

## üé≤ Maze Generation Algorithms

Each generator creates a *perfect maze* ‚Äî meaning one unique path exists between any two points.

1. **DFS Backtracker** ‚Äì Depth-first traversal, produces twisty paths
2. **Prim‚Äôs Algorithm** ‚Äì Builds out from walls, yields dense, branching mazes
3. **Wilson‚Äôs Algorithm** ‚Äì Loop-erased random walks, yields uniform randomness
4. **Recursive Division** ‚Äì Carves passages through subdivided regions
5. **Hunt and Kill** ‚Äì Alternates random walks with deterministic scanning
6. **Kruskal‚Äôs Algorithm** ‚Äì Uses a randomized spanning tree, very natural structure

## üöÄ Maze Solving Algorithms

Solvers animate their search and highlight the final path in green.

1. **A\*** ‚Äì Efficient heuristic search using both path cost and estimated distance
2. **Dijkstra** ‚Äì Uniform-cost search for optimal paths in weighted graphs
3. **Breadth-First Search (BFS)** ‚Äì Explores all nodes at one level before moving deeper
4. **Greedy Best-First Search** ‚Äì Always chooses the node closest to the goal
5. **Depth-First Search (DFS)** ‚Äì Explores as far as possible along each path
6. **Bidirectional Search** ‚Äì Simultaneous search from start and goal

## üß™ Usage

**Requirements:**

```bash
pygame>=2.0
numpy>=1.21
```

**Installation**

```bash
git clone git@github.com:Drew-Woodz/MazeSolver.git
cd MazeSolver
python -m venv .venv
source .venv/Scripts/activate  # On Windows
# or
source .venv/bin/activate      # On Mac/Linux

pip install pygame numpy
```

**Run from Command Line**

```bash
python main.py --algo 2 --size 30 --solve 3
```

**Arguments:**

* `--algo: Maze generator (1‚Äì6)`
* `--size: Maze dimensions (NxN)`
* `--solve: Solver algorithm (1‚Äì6) (optional)`

**Examples:**

```bash
python main.py --algo 4 --size 40 --solve 1
```
